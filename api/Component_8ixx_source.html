14 /**
15 * @brief Abstract base class for components attached to GameObjects.
16 *
17 * @details Components encapsulate specific behaviors or data that can be attached
18 * to a GameObject. Each component maintains a back-reference to its owning
19 * GameObject, established via onAttach().
20 *
21 * Derived classes implement specific functionality such as physics, rendering,
22 * or input handling. Components can optionally implement the Updatable interface
23 * if they require per-frame updates.
24 *
25 * Example usage:
26 * ```cpp
27 * class HealthComponent : public helios::engine::game::Component {
28 * int health_ = 100;
29 * public:
30 * void takeDamage(int amount) { health_ -= amount; }
31 * int health() const { return health_; }
32 * };
33 *
34 * // Attach to a GameObject

36 * ```
