17 /**
18 * @brief Abstract base class for components attached to GameObjects.
19 *
20 * @details Components encapsulate specific behaviors or data that can be attached
21 * to a GameObject. Each component maintains a back-reference to its owning
22 * GameObject, established via onAttach().
23 *
24 * Derived classes implement specific functionality such as physics, rendering,
25 * or input handling. Components can optionally implement the Updatable interface
26 * if they require per-frame updates.
27 *
28 * ## Enabled State
29 *
30 * Each component has an `isEnabled()` flag that controls whether it participates
31 * in system processing. This allows fine-grained control over individual behaviors
32 * without removing components from the GameObject.
33 *
34 * - `isEnabled() == true` — Component is active and processed by systems
35 * - `isEnabled() == false` — Component is skipped by systems but remains attached
36 *
37 * Systems should check `isEnabled()` before processing a component:
38 *
39 * ```cpp
40 * for (auto [obj, collision] : gameWorld.find<CollisionComponent>().each()) {
41 * if (collision.isDisabled()) {
42 * continue; // Skip disabled components
43 * }
44 * // Process enabled component...
45 * }
46 * ```
47 *
48 * @note Disabling a component does **not** disable the entire GameObject.
49 * Use `GameObject::setActive(false)` to exclude an entity from all processing.
50 *
51 * Example usage:
52 * ```cpp
53 * class HealthComponent : public helios::engine::ecs::Component {
54 * int health_ = 100;
55 * public:
56 * void takeDamage(int amount) { health_ -= amount; }
57 * int health() const { return health_; }
58 * };
59 *
60 * // Attach to a GameObject
61 * auto& health = gameObject.add<HealthComponent>();
62 *
63 * // Temporarily disable collision
64 * gameObject.get<CollisionComponent>()->disable();
65 * ```
66 *
67 * @see GameObject
68 * @see CloneableComponent
69 */
