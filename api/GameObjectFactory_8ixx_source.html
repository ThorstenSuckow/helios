22 /**
23 * @brief Factory class for mass-producing GameObjects from a prefab.
24 *
25 * @details GameObjectFactory holds a prototype (prefab) of a GameObject and provides
26 * methods to fill a GameObjectPool with clones of that prefab. This enables efficient
27 * spawning by pre-allocating objects and reusing them via pooling.
28 *
29 * The factory owns the prefab exclusively and clones it into the GameWorld as needed.
30 * Created objects are immediately marked inactive and registered with the pool.
31 *
32 * Example usage:
33 * ```cpp
34 * auto prefab = std::make_unique<GameObject>();
35 * prefab->add<RenderableComponent>(bulletMesh);
36 * prefab->add<Move2DComponent>();
37 *
38 * GameObjectFactory factory(std::move(prefab));
39 * factory.fillPool(gameWorld, bulletPool);
40 * ```

75 *
76 * @details Calculates the remaining capacity in the pool and creates that many
77 * clones of the prefab. Each clone is:
78 * - Added to the GameWorld via `clone()`
79 * - Marked as inactive (`setActive(false)`)
80 * - Registered with the pool's inactive list
81 *
82 * This method is typically called once during initialization to pre-populate

85 * @param gameWorld The GameWorld that will own the cloned GameObjects.
86 * @param gameObjectPool The pool to fill with inactive clones.

90 helios::engine::runtime::pooling::GameObjectPool& gameObjectPool
91 ) {
92 const size_t used = gameObjectPool.activeCount() + gameObjectPool.inactiveCount();
93 const size_t space = used < gameObjectPool.size() ? gameObjectPool.size() - used : 0;
94
95 for (size_t i = 0; i < space; i++) {
96 helios::engine::ecs::GameObject* go = gameWorld.clone(*gameObjectPrefab_);
97 if (go) {
98 go->setActive(false);
