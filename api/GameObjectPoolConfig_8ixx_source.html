16 /**
17 * @brief Configuration structure for creating a GameObjectPool.
18 *
19 * @details GameObjectPoolConfig bundles all information required to initialize
20 * a GameObjectPool: a unique pool identifier, a PrefabId referencing the
21 * prefab template, and the initial pool size.
22 *
23 * The PrefabId identifies the prefab GameObject that serves as the archetype
24 * from which all pooled instances are cloned. The prefab is registered
25 * separately via GameObjectFactory::withPrefabId() and looked up by the
26 * pool system at initialization time.
27 *
28 * Example usage:
29 * ```cpp
30 * // Create and tag prefab
31 * auto enemyPrefab = GameObjectFactory::instance()
32 * .gameObject(gameWorld)
33 * .withPrefabId(EnemyPrefabId)
34 * // ... component configuration ...
35 * .make();
36 *
37 * auto config = std::make_unique<GameObjectPoolConfig>(
38 * EnemyPoolId,
39 * EnemyPrefabId,
40 * 50 // Pre-allocate 50 clones
41 * );
42 *
43 * poolManager.addPoolConfig(std::move(config));
44 * ```
45 *
46 * @see PrefabId
47 * @see PrefabIdComponent
48 * @see GameObjectPool
