22 /**
23 * @brief System that evaluates spawn conditions and enqueues spawn commands.
24 *
25 * @details GameObjectSpawnSystem bridges the spawn scheduling logic with the
26 * command-based spawning pipeline. It performs the following each frame:
27 *
28 * 1. **Read Frame Events:** Consumes `SpawnPlanRequestExecutedEvent` from the
29 * previous frame to commit completed spawn operations to the scheduler.
30 *
31 * 2. **Evaluate Conditions:** Calls `SpawnScheduler::evaluate()` to check all
32 * registered spawn rules against their conditions (timer, trigger, etc.).
33 *
34 * 3. **Drain Scheduled Plans:** Retrieves all plans that are ready to execute
35 * and enqueues them as `ScheduledSpawnPlanCommand` into the CommandBuffer.
36 *
37 * The actual spawning is performed by the command dispatcher/manager layer,
38 * keeping this system focused on scheduling logic only.
39 *
40 * Example setup:
41 * ```cpp
42 * auto scheduler = std::make_unique<SpawnScheduler>();
43 * scheduler->addRule(enemyRuleId, enemyCondition, enemyProfileId);
44 *
45 * auto spawnSystem = std::make_unique<GameObjectSpawnSystem>(
46 * std::move(scheduler)
47 * );
48 * mainPhase.addPass().add(std::move(spawnSystem));
49 * ```
50 *
51 * @note This system reads frame-level events via `readFrame()`, meaning it
52 * processes spawn confirmations from the previous frame.
53 *
54 * @see SpawnScheduler
55 * @see ScheduledSpawnPlanCommand
56 * @see SpawnPlanRequestExecutedEvent
57 */
