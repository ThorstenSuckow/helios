42 /**
43 * @brief System that evaluates spawn conditions and emits SpawnCommands.
44 *
45 * @details GameObjectSpawnSystem integrates spawning into the game loop by
46 * evaluating a SpawnCondition each frame to determine how many objects should
47 * be spawned. When the spawn budget is greater than zero, it emits a SpawnCommand
48 * to the CommandBuffer for deferred execution.
49 *
50 * The actual spawning is handled by the SpawnCommandDispatcher and SpawnManager
51 * during the command flush phase.
52 *
53 * Example usage:
54 * ```cpp
55 * auto condition = std::make_unique<TimerSpawnCondition>(2.0f);
56 *
57 * gameLoop.phase(PhaseType::Main)
58 * .addPass()
59 * .addSystem<GameObjectSpawnSystem>(poolId, std::move(condition));
60 * ```
61 *
62 * @see SpawnCondition
63 * @see SpawnCommand
64 * @see SpawnManager
65 */
