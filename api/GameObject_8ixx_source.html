16 /**
17 * @brief Base class representing an interactive game entity in the game world.
18 *
19 * @details The `GameObject` provides a game-logic-level abstraction over a `SceneNode`.
20 * It serves as the primary building block for game-specific entities like players or enemies,
21 * designed to be subclassed to add further behavior. Each `GameObject` has a unique
22 * `Guid` for easy identification and provides convenient methods for manipulating the
23 * transform of its underlying `SceneNode`. It also acts as the target for `Command` execution.
24 *
25 * Example usage:
26 * ```cpp
27 * class Player : public helios::game::GameObject {
28 * public:
29 * using GameObject::GameObject;
30 *
31 * void jump() {
32 * translate({0.0f, 5.0f, 0.0f});
33 * }
34 * };
35 *
36 * auto sceneNode = std::make_shared<helios::scene::SceneNode>();
37 * auto player = std::make_unique<Player>(sceneNode.get());
38 * player->translate({1.0f, 0.0f, 0.0f});
39 * ```
40 *
41 * @warning The GameObject does NOT own the SceneNode. The caller must ensure the SceneNode
42 * outlives the GameObject. If the SceneNode is destroyed while the GameObject is
43 * still alive, all transform operations will result in undefined behavior.
44 *
45 * @todo Implement observer pattern or weak_ptr to detect SceneNode destruction and
46 * invalidate the transform pointer to prevent dangling pointer issues.
47 */
