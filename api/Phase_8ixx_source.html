45 /**
46 * @brief Represents a phase in the game loop containing multiple passes.
47 *
48 * A Phase groups related systems into passes that are executed sequentially.
49 * Each pass can contain multiple systems and may have a commit point for
50 * event synchronization.
51 *
52 * Commit points allow fine-grained control over when commands are flushed,
53 * managers are processed, and pass-level events are synchronized. See
54 * CommitPoint for available synchronization options.
55 *
56 * Phases are owned by the GameLoop and should not be created directly.
57 *
58 * @see Pass
59 * @see CommitPoint
60 * @see GameLoop
61 */
