16 /**
17 * @brief Component that identifies which pool a GameObject belongs to.
18 *
19 * @details PoolIdComponent stores the GameObjectPoolId of the pool that
20 * owns this entity. This is essential for:
21 *
22 * - **Despawn Routing:** When a pooled entity needs to be returned, the
23 * PoolIdComponent tells the system which pool's handler to notify.
24 * - **Pool Membership:** Identifies entities that belong to a specific pool.
25 *
26 * The component is cloneable, so pooled object templates can be duplicated
27 * while preserving their pool association.
28 *
29 * Example usage:
30 * ```cpp
31 * // During pool population
32 * gameObject->add<PoolIdComponent>(bulletPoolId);
33 *
34 * // During despawn
35 * auto* pic = gameObject->get<PoolIdComponent>();
36 * if (pic) {
37 * auto handler = gameWorld.poolRequestHandler(pic->gameObjectPoolId());
38 * handler->submit(DespawnRequest{gameObject->guid(), pic->gameObjectPoolId()});
39 * }
40 * ```
41 *
42 * @see GameObjectPool
43 * @see GameObjectPoolId
44 * @see PoolRequestHandler
45 */
