23 /**
24 * @brief Encapsulates a single rendering pass with its associated resources.
25 *
26 * A `RenderPass` holds a `RenderQueue` containing all `RenderCommand` and
27 * `TextRenderCommand` objects to be processed. It also stores frame-level
28 * uniform values (e.g., view and projection matrices) that remain constant
29 * during the pass.
30 *
31 * ## Components
32 *
33 * - **RenderQueue:** Contains geometry and text render commands.
34 * - **Viewport:** Defines the rendering area and clear settings.
35 * - **UniformValueMap:** Frame-level uniforms applied to all commands.
36 *
37 * ## Lifecycle
38 *
39 * ```
40 * RenderingDevice::beginRenderPass(pass)
41 * → Clear buffers, configure viewport
42 * RenderingDevice::doRender(pass)
43 * → Process RenderCommands and TextRenderCommands
44 * RenderingDevice::endRenderPass(pass)
45 * → Finalize pass, unbind resources
46 * ```
47 *
48 * @note Future versions should support additional configuration like depth
49 * testing, stencil operations, and draw modes.
50 *
51 * @see RenderQueue
52 * @see RenderingDevice
53 * @see Viewport
54 */
