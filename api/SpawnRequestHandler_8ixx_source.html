15 /**
16 * @brief Interface for handlers that process spawn and despawn requests.
17 *
18 * @details SpawnRequestHandler defines the contract for managers that handle
19 * entity lifecycle operations for GameObjectPools. Each handler is registered
20 * with a SpawnProfileId via `GameWorld::registerSpawnRequestHandler()`.
21 *
22 * When a pooled entity needs to spawn or despawn, the appropriate handler
23 * is looked up by profile ID and receives the request for deferred processing.
24 *
25 * ## Request Types
26 *
27 * - **SpawnRequest:** Requests acquisition of an entity from the pool
28 * - **DespawnRequest:** Requests return of an entity to the pool
29 * - **ScheduledSpawnPlanRequest:** Requests batch spawning from scheduler
30 *
31 * ## Implementation Notes
32 *
33 * Handlers typically also implement the Manager interface to receive flush()
34 * calls for deferred request processing during the manager flush phase.
35 *
36 * Example:
37 * ```cpp
38 * class MyPoolManager : public Manager, public SpawnRequestHandler {
39 * bool submit(SpawnRequest req) noexcept override {
40 * spawnQueue_.push(req);
41 * return true;
42 * }
43 * bool submit(DespawnRequest req) noexcept override {
44 * despawnQueue_.push(req);
45 * return true;
46 * }
47 * bool submit(ScheduledSpawnPlanRequest req) noexcept override {
48 * planQueue_.push(req);
49 * return true;
50 * }
51 * void flush(GameWorld& world, UpdateContext& ctx) noexcept override {
52 * // Process queued requests
53 * }
54 * };
55 * ```
56 *
57 * @see SpawnRequest
58 * @see DespawnRequest
59 * @see ScheduledSpawnPlanRequest
60 * @see SpawnManager
61 * @see PoolManagerRegistry
62 */
