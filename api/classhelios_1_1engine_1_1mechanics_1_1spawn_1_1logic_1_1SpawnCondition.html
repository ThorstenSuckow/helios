Detailed Description

Abstract base class defining when and how many GameObjects should spawn.

SpawnCondition implements the Strategy pattern for spawn timing control. Concrete implementations determine how many objects should be spawned each frame based on time, game state, pool availability, or other conditions.

The spawn budget returned by `spawnBudget()` represents the number of GameObjects that the spawn system should attempt to activate this frame.

Calculates how many GameObjects should be spawned this frame.

Called each frame by the spawn system to determine how many objects to activate from the pool. The implementation may consider elapsed time, pool capacity, game state, or any other relevant factors.

Parameters
gameWorldPointer to the current GameWorld. May be nullptr.
gameObjectPoolPointer to the pool from which objects will be acquired.
updateContextThe current frame's update context with delta time.
Returns
The number of GameObjects to spawn this frame (spawn budget). Returns 0 if no spawning should occur.
