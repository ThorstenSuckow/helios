Detailed Description

Abstract base class for game systems.

Systems are global game logic processors that operate on the GameWorld rather than individual GameObjects — ideal for cross-cutting concerns like physics, collision detection, or object pooling.

Concrete systems are plain classes that provide `update(UpdateContext&)` (required) and optionally `init(GameWorld&)`. They are wrapped automatically by TypedSystem<T> when registered via SystemRegistry.
