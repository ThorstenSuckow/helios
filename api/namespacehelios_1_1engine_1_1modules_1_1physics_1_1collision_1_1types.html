enum class helios::engine::modules::physics::collision::types::CollisionBehavior : uint8_t
exportstrong

Defines how an entity responds to a collision event.

CollisionBehavior values are used as bitmasks to configure per-layer collision responses in CollisionComponent. Multiple behaviors can be combined using bitwise operations.

The behavior determines what action the collision system takes when two entities collide based on their layer configuration.

Note
Not all behaviors are currently fully implemented. Currently supported:
  • `Reflect` — Fully supported
  • `Bounce` — Fully supported
  • `Despawn` — Fully supported
Enumerator
None 

No collision response.

Reflect 

Reflect the entity's velocity off the collision surface.

Uses the surface normal to calculate a reflected velocity vector. Commonly used for projectiles bouncing off walls.

Bounce 

Bounce the entity with restitution applied.

Similar to Reflect but applies a restitution coefficient to reduce velocity. Used for physics-based bouncing.

Stick 

Stick the entity to the collision surface.

Stops the entity's movement and attaches it to the collided object. Used for projectiles that embed on impact.

Despawn 

Despawn the entity on collision.

Marks the entity for removal from the game world. Used for projectiles that are destroyed on impact.

Event 

Emit an event without physical response.

Generates a collision event for game logic processing without affecting the entity's physics state.

size_ 

Internal: Number of behavior types for array sizing.

Warning
Do not use as a behavior value.
