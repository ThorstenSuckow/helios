Description​

Central registry for managing game entities, systems, and the active level.

The GameWorld is the root container for the game state. It manages the lifecycle of GameObjects, executes Systems, and holds the current Level.

Key responsibilities:

  • **Entity Management:** Owns all GameObjects and provides lookup by Guid.
  • **System Execution:** Manages and updates registered Systems.
  • **Level Management:** Holds the active Level instance.
  • **Update Loop:** Orchestrates the frame update by updating GameObjects and then Systems.

GameObjectRange​

template <class MapT, class... Cs>
using helios::engine::game::GameWorld::GameObjectRange = GameObjectView<MapT, helios::engine::game::GameObject, Cs...>

Lazy range type for component-filtered GameObject iteration.

This alias wraps GameObjectView with the appropriate map and component types. The view filters GameObjects on-the-fly during iteration, yielding only those that possess all specified component types.

info

GameObjectView is included as a .h header in the global module fragment due to MSVC ICE issues when used as a .ixx module interface unit.

Template Parameters
MapT

The map type (const or non-const).

Cs

The component types to filter by.

find()​

template <class... Cs>
auto helios::engine::game::GameWorld::find (GameObjectFilter query=GameObjectFilter::Active|GameObjectFilter::ComponentEnabled)
 inline nodiscard

Finds all GameObjects that have the specified component types.

Returns a lazy range that filters GameObjects on-the-fly during iteration. Only GameObjects possessing all specified component types are yielded.

Example usage: ```cpp // Range-based for loop for (auto* obj : gameWorld.find<Move2DComponent, SceneNodeComponent>()) { auto* move = obj->get<Move2DComponent>(); move->setVelocity({1.0f, 0.0f, 0.0f}); }

// Using each() for direct component access (structured bindings) for (auto [obj, move] : gameWorld.find<Move2DComponent>().each()) { move.get().setVelocity({1.0f, 0.0f, 0.0f}); } ```

Template Parameters
Cs

The component types to filter by. GameObjects must have all types.

Returns

A GameObjectView object that can be iterated over.

update()​

void helios::engine::game::GameWorld::update (helios::engine::game::UpdateContext & updateContext)
 noexcept

Updates the game world state for the current frame.

This method performs the following steps: 1. Updates all active GameObjects (invoking their `update()` method). 2. Updates all registered Systems (invoking their `update()` method).

Parameters
updateContext

Context containing deltaTime, input snapshot, game world and command buffer.

info

The order of GameObject updates is not guaranteed. Systems are updated in registration order.
