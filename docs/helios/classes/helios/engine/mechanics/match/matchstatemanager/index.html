Description​

Manages match state transitions using a rule-based state machine.

The MatchStateManager processes MatchStateCommands and applies transition rules to move between match states (e.g., Warmup -> PlayerSpawn -> Playing). It supports guard callbacks for conditional transitions and notifies registered listeners on state changes.

## Transition Flow

1. Commands are submitted via `submit()` and queued 2. On `flush()`, the last pending command is evaluated against rules 3. If a rule matches and its guard passes, the transition executes:

  • `onMatchStateExit()` is called on listeners
  • `onMatchStateTransition()` is called on listeners
  • Session state is updated
  • `onMatchStateEnter()` is called on listeners

Processes pending state commands and applies valid transitions.

Evaluates the last pending command against the configured rules. If a matching rule is found and its guard (if any) passes, the transition is applied and listeners are notified.

184 void flush(
185 helios::engine::runtime::world::GameWorld& gameWorld,
186 helios::engine::runtime::world::UpdateContext& updateContext
187 ) noexcept override {
188
189 if (pending_.empty()) {
190 return;
191 }
192
193 auto command = pending_.back();
194
195 auto transitionRequest = command.transitionRequest();
196
197 auto& session = gameWorld.session();
198 auto currentFrom = session.matchState();
199 auto from = transitionRequest.from();
200 auto transitionId = transitionRequest.transitionId();
201
202
203 if (currentFrom != from) {
204 pending_.clear();
205 return;
206 }
207
208
209 for (auto& rule : rules_) {
210 if (rule.from() == from && rule.transitionId() == transitionId) {
211
212 // invoke the transitions that can satisfy the guard
213 if (rule.guard()) {
214 // if the guard does not evaluate to true, do not change the state
215 if (!rule.guard()(updateContext, transitionRequest)) {
216 break;
217 }
218 }
219
220 // clean up
221 signalExit(from, rule.to(), transitionId, updateContext);
222 signalTransition(from, rule.to(), transitionId, updateContext);
223 session.setMatchStateFrom(MatchStateTransitionContext(currentFrom, rule.to(), transitionId));
224 signalEnter(from, rule.to(), transitionId, updateContext);
225 }
226 }

228 pending_.clear();
