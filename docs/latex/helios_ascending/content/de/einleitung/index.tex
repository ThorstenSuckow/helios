\chapter{Einleitung}

Der im Rahmen des Projektstudiums im Fernstudium Informatik zu erstellende \textit{Geometry Wars}-Klon ist in~\cite{SH25} beschrieben.
Im Folgenden werden die wesentlichen Anforderungen und Spezifikationen erläutert, um diese im späteren Verlauf referenzieren und vergleichen zu können.\\

\noindent
Als zu realisierende Features wurden vereinbart:

\begin{itemize}
    \itemsep0.5em
    \item Spielbares Level (2D-Grid)
    \item Time-Attack-Modus mit einer Dauer von 3 Minuten
    \item Highscore-System
    \item Controller-Steuerung
    \item Stabile Framerate von mindestens 60 FPS bei gleichzeitiger Darstellung von über hundert Gegnern und Objekten
    \item Drei Gegnertypen mit einfacher KI (``Spawn and Chase``)
    \item Aufwertung des \textit{Game Feels} durch grafische Effekte
    \item Kompilierbar und lauffähig unter Windows 11
\end{itemize}

\noindent
Zum Zeitpunkt der Veröffentlichung des oben genannten Dokumentes war bereits Meilenstein 1 erreicht, welcher den Fokus auf das Anwendungsgerüst legte.
In~\ref{tab:zeitplan} ist eine Übersicht der nachfolgenden Meilensteine 1--6 dargestellt. Deren Fertigstellungstermine wurden mit einer jeweiligen Verzögerung von zwei bis vier Wochen erreicht.
Hierbei zeigte sich, dass die zunehmende Komplexität der Features - wie das Input-System, die Spawn-Logik, die Kollisionsabfrage und das UI-Rendering - wiederholt Anpassungen an bereits existierenden Systemen erforderten.

So erforderte beispielsweise das Input-System die Berücksichtigung von Stick-Drift; das Spawning von Gegnerobjekten machte einen Pool wiederverwendbarer Objekte (\textit{prefabricated objects}, i.F. \textit{prefab}) notwendig.
Zusätzlich zur reinen Kollisionserkennung ergaben sich komplexe Fragestellungen hinsichtlich der Verarbeitung dieser Kollisionen innerhalb der beteiligten Systeme.
Das UI-Rendering bedingte eine Erweiterung der Rendering-Engine um die Unterstützung von Texturen und UI-Elementen sowie die Implementierung hierarchischer Zustandsautomaten (HFSM), um die verschiedenen Spielzustände (z.\,B. ``Main Menu``, ``In Game``, ``Game Over``) sauber zu kapseln.
